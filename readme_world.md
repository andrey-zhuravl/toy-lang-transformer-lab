# Руководство по генерации "игрушечного" мира

## Обзор

Этот документ описывает основные принципы и последовательность шагов при работе с симулятором мира, добавленным в проект. Под "миром" понимается модуль, который моделирует состояние набора сущностей (агенты, предметы, локации), применяет действия к состоянию и генерирует датасеты для обучения моделей.

## 1. Проектирование онтологии

1. Определите типы сущностей (`EntityType`), отношений (`RelationType`) и действий.
2. Для каждого действия задайте сигнатуру ролей, предусловия (`pre`) и эффекты (`eff`). Они описываются в конфигурационном файле `src/world/world_config.yaml`.
3. Убедитесь, что эффекты не противоречат друг другу и что предусловия проверяемы по текущему состоянию.

## 2. Настройка конфигурации мира

1. Откройте файл `src/world/world_config.yaml` и задайте параметры окружения: количество агентов, предметов и локаций, длину эпизода, режим генерации имён и т.д.
2. При необходимости добавьте дополнительные действия или измените существующие, следуя формату:
   ```yaml
   - name: идти
     schema: [АГЕНТ, ЛОКАЦИЯ]
     pre: []
     eff: ["loc(АГЕНТ)=ЛОКАЦИЯ"]
   ```
3. Настройте секцию `render`, чтобы выбрать стиль вербализации и дополнительные опции (например, кавычки для спанов).

## 3. Интеграция в общий конфиг генерации

1. В `src/data_generation/config.yaml` убедитесь, что включён режим `world_mode: true` и указан путь к файлу мира:
   ```yaml
   world_config: "src/world/world_config.yaml"
   ```
2. Выберите набор задач в секции `tasks` (QA, парсинг, семантика → NL, предсказание эффектов, LM).
3. Настройте размеры датасета, сплиты и возможные ограничения (holdout по объектам, длине эпизодов и т.д.).

## 4. Генерация эпизодов

Эпизоды описываются в `src/world/episode.py`. Они строятся так:

1. Случайным образом выбираются сущности и их начальные свойства.
2. Симулятор (`src/world/simulator.py`) пошагово применяет допустимые действия, проверяя предусловия и обновляя состояние.
3. Лог эпизода содержит триаду `(действие, состояние_до, состояние_после)` для каждого шага.

## 5. Рендеринг и запросы

1. Модули `src/world/renderers/nl_renderer.py` и `src/world/renderers/sem_renderer.py` преобразуют состояния и действия в текст и семантическое представление.
2. `src/world/queries.py` строит вопросы к состоянию/истории и ответы для задач QA.

## 6. Построение датасетов

Используйте модуль `src/data_generation/world_tasks.py` или CLI для формирования JSONL-файлов:

- `world_qa.jsonl` — вопросы по текущему или итоговому состоянию.
- `world_parsing.jsonl` — преобразование NL → Sem.
- `world_sem2nl.jsonl` — обратная задача Sem → NL.
- `world_effect.jsonl` — прогноз последствий действия.
- (Опционально) `world_log_lm.jsonl` — языковая модель на последовательности шагов.

Каждый элемент содержит поля `input`, `output` и `meta`.

## 7. CLI для генерации

Запустите генерацию через `world_cli`:

```bash
python -m src.cli.world_cli generate \
  --world-config src/world/world_config.yaml \
  --data-config src/data_generation/config.yaml \
  --out-dir data/datasets/ \
  --episodes 1000 \
  --export all \
  --seed 42
```

Ключи:
- `--episodes` — количество сгенерированных эпизодов.
- `--export` — список задач (`qa`, `parsing`, `sem2nl`, `effect`, `lm`, `all`).
- `--seed` — фиксированное зерно для воспроизводимости.

Во время выполнения команда просканирует сгенерированные JSONL-файлы и соберёт словарь токенов, появляющихся во входах и выходах задач. Файл словаря будет сохранён в `data/vocab.json` (или в путь, указанный в `vocab_path` конфигурации).

Если нужно пересобрать словарь после дополнительной фильтрации/объединения датасетов, используйте подкоманду `build-vocab`, передав путь к манифесту:

```bash
python -m src.cli.world_cli build-vocab \
  --manifest data/datasets/manifest_world.json \
  --output data/vocab.json
```

Если аргумент `--output` не указан, файл будет записан по пути `vocab_path`, указанному в манифесте, или рядом с ним.

## 8. Проверка качества

1. Убедитесь, что симулятор не формирует логических противоречий (валидатор в `simulator.py`).
2. Проверьте размеры и целостность датасетов (например, с помощью `jq` или кастомных скриптов).
3. Используйте существующие утилиты анализа (раздел `src/analysis/`) для оценки метрик и визуализации внимания моделей.

## 9. Рекомендации по расширению

- Добавляйте новые типы сущностей, отношений и действий через расширение онтологии и конфигурации.
- Вводите новые типы задач в `world_tasks.py`, если требуется иной формат данных.
- Для экспериментов с OOV/копированием используйте режимы имён (`names_mode`) и секцию `holdout` в конфиге генерации.

Следуя этим шагам, можно от настройки мира перейти к созданию полноценного набора датасетов для обучения и анализа трансформеров.
